ptfrepos - how L3 is going to release PTFs into custom repositories

How we are doing right now?

  • rpms in an URL
  • download each rpm
  • call zypper with all the required rpms
  • fail to do so and ask quesitons

What challenges our users face?

  • huge server park, thousands of SUSE servers
  • they are running with many PTFs
  • dislike for our current delivery method

Where we want to get?

  • multiple products per namespace
  • multiple namespaces per customer
  • automatic provisioning on registration
